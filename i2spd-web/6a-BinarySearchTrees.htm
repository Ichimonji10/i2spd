Binary Search Trees Module Help Learn more.

In this module, we discover a new self-referential form of data, the binary search tree. We will investigate how the structure of data affects performance, especially when it comes to search time.

Additionally, we learn to create data definitions of and design functions on binary search trees, though we'll find this is largely just an application of the template rules and recipes we've learned so far.

The material in this module should take approximately 5-6 hours of dedicated time to complete, including working along with the lecture videos, doing the practice problems, doing the homework problems and the short quiz.

Learning Goals

  • Be able to identify problem domain information of arbitrary size that should be represented using binary search trees.
  • Be able to use the design recipes to design with binary search trees.

Tips for Success

The programs get a bit longer again in this module, but most of the work is with existing recipe skills. To avoid feeling overwhelmed its really important to keep track of what it is you need to do NOW. Don't think about the whole problem at once - think about the current recipe step now and do the rest later.

Trust the natural recursion!
